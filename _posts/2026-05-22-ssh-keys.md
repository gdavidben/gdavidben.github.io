## Generate a new SSH key pair

{% highlight bash linenos %}
ssh-keygen -t ed25519 -C "your_email@example.com"
{% endhighlight %}

**What it does**: Creates a new public/private key pair using the Ed25519 algorithm. You will be prompted to choose a file location (default `~/.ssh/id_ed25519`) and a passphrase.

If you need RSA for compatibility with older systems:

{% highlight bash linenos %}
ssh-keygen -t rsa -b 4096 -C "your_email@example.com"
{% endhighlight %}

After running the command, two files are created:

- `~/.ssh/id_ed25519` — your **private** key (never share this)
- `~/.ssh/id_ed25519.pub` — your **public** key (safe to share)


## View your public key

{% highlight bash linenos %}
cat ~/.ssh/id_ed25519.pub
{% endhighlight %}

**What it does**: Prints the public key content so you can copy it to GitHub, GitLab, a remote server, etc.


## Change the passphrase of an existing private key

{% highlight bash linenos %}
ssh-keygen -p -f ~/.ssh/id_ed25519
{% endhighlight %}

**What it does**: Prompts you for the current passphrase, then asks for a new one. The private key file is updated in place — no new key pair is generated, so your public key remains the same.


## Save the passphrase in the macOS Keychain

### Step 1 — Add the key to the SSH agent with Keychain storage

{% highlight bash linenos %}
ssh-add --apple-use-keychain ~/.ssh/id_ed25519
{% endhighlight %}

**What it does**: Adds the private key to the SSH agent and stores the passphrase in the macOS Keychain. You only need to type the passphrase once.

### Step 2 — Configure SSH to use the Keychain automatically

Add the following to `~/.ssh/config` (create the file if it doesn't exist):

{% highlight text linenos %}
Host *
  AddKeysToAgent yes
  UseKeychain yes
  IdentityFile ~/.ssh/id_ed25519
{% endhighlight %}

**What it does**: Tells SSH to automatically load keys into the agent and retrieve passphrases from the Keychain on every connection, so you never have to type the passphrase again — even after a reboot.


## Verify the key is loaded in the agent

{% highlight bash linenos %}
ssh-add -l
{% endhighlight %}

**What it does**: Lists all keys currently loaded in the SSH agent. You should see your key fingerprint in the output.


## Test the connection (GitHub example)

{% highlight bash linenos %}
ssh -T git@github.com
{% endhighlight %}

**What it does**: Tests the SSH connection to GitHub. If everything is configured correctly, you'll see a message like `Hi username! You've successfully authenticated`.
